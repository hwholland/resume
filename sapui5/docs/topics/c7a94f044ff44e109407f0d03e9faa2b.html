Creating a KPI Card

KPI card is a smart component that uses UI annotations to render KPI content. It is comprised of a header and content area.

Hover over each action for a description. Click the action for more information.

Descriptor Settings: KPI Card

Property: keyPerformanceIndicators

Description: .

Configuration Sample:
"sap.ui.generic.app":{  
   "pages":[  
      {  
         "entitySet":"SEPMRA_C_ALP_SlsOrdItemCube",
         "component":{  
            "name":"sap.suite.ui.generic.template.AnalyticalListPage",
            "list":true,
            "settings":{  
               "keyPerformanceIndicators":{  
                  "KPIRevenue":{  
                     "model":"kpi",
                     "entitySet":"SEPMRA_C_ALP_TotalSalesKPI",
                     "qualifier":"KPIRevenue",
                     "detailNavigation":"OverviewPage"
                  }
            }
      }
},

Annotation: SelectionPresentationVariant with Qualifier="KPIRevenue"

Configuration Sample:

The SelectionPresentationVariant annotation qualifier relates to a PresentationVariant that provides a DataPoint annotation (for the KPI value).

Configuration Sample:

The SelectionPresentationVariant annotation qualifier relates to a SelectionVariant (optional) and PresentationVariant that provides a DataPoint and Chart annotations. 

</Annotation>
  <Annotation Term="UI.SelectionPresentationVariant" Qualifier="KPIRevenue">
    <Record>
      <PropertyValue Property="Text" String="KPI: Net Sales by Calendar Quarter" />
      <PropertyValue Property="SelectionVariant" Path="@UI.SelectionVariant#KPIRevenue" />
      <PropertyValue Property="PresentationVariant" Path="@UI.PresentationVariant#KPIRevenue" />
    </Record>
</Annotation>
Sample SelectionVariant:
<Annotation Term="UI.SelectionVariant" Qualifier="KPIRevenue">
  <Record>
    <PropertyValue Property="Text" String="Default" />
  </Record>
</Annotation>

Annotation: DataPoint

Description: Configure this annotation to display KPI name, KPI value, scale factor, number of fractional digits, and criticality. 

<Annotation Term="UI.DataPoint" Qualifier="KPIValue">
  <Record>
    <PropertyValue Property="Value" Path="KPIValue" />
    <PropertyValue Property="Title" String="Sales Revenue" />
    <PropertyValue Property="Description" String="Sales revenue in the current quarter" />
    <PropertyValue Property="TargetValue" Path="KPITargetValue" />
    <PropertyValue Property="Visualization" EnumMember="UI.VisualizationType/Number" />
    <PropertyValue Property="ValueFormat">
      <Record>
        <PropertyValue Property="ScaleFactor" Decimal="1000000" />
        <PropertyValue Property="NumberOfFractionalDigits" Int="2" />
      </Record>
    </PropertyValue>
	<PropertyValue Property="Criticality" EnumMember="com.sap.vocabularies.UI.v1.CriticalityType/Positive"/>
        <PropertyValue Property="CriticalityCalculation">
          <Record>
            <PropertyValue Property="ImprovementDirection" EnumMember="UI.ImprovementDirectionType/Maximize" />
            <PropertyValue Property="DeviationRangeLowValue" Path="KPIDeviationRangeLowValue" />
            <PropertyValue Property="ToleranceRangeLowValue" Path="KPIToleranceRangeLowValue" />
          </Record>
        </PropertyValue>
        <PropertyValue Property="TrendCalculation">
          <Record>
            <PropertyValue Property="ReferenceValue" Path="KPITargetValue" />
            <PropertyValue Property="IsRelativeDifference" Bool="true" />
            <PropertyValue Property="UpDifference" Decimal="0.1" />
            <PropertyValue Property="StrongUpDifference" Decimal="1" />
            <PropertyValue Property="DownDifference" Decimal="0.1" />
            <PropertyValue Property="StrongDownDifference" Decimal="1" />
           </Record>
        </PropertyValue>
  </Record>
</Annotation>
In the header area you can view aggregated value of a key performance indicator (KPI), the trend, and percentage of change. In the chart area you can view graphical representation of data.
Note If you configure the Analytics card header, chart descriptions (x- and y-axis) are not displayed in the card (except for bubble chart). For example, the chart title Net Sales by Days Payable, already conveys that the y-axis is Net Sales and the x-axis represents Days Payable.

Chart Responsiveness

To improve the chart response you can use UI.PresentationVariant.MaxItems to limit the number of records fetched from backend that are rendered on the screen. For example:

Sample Code 
<Annotation Term="UI.PresentationVariant" Qualifier="Column_Eval_by_Country_123">
    <Record>      
          <PropertyValue Property="MaxItems" Int="5" />
    </Record>
</Annotation>
Tip If there are too many data records displayed on the chart, it is difficult to select a data point. If you are only interested in checking the top records, then use UI.PresentationVariant.SortOrder. For more information, see Configuring Card Properties.

Axis Scaling

Axis Scaling lets you to scale and display data for line, bubble or scatter charts in the analytics card. You can choose to use any of the following Axis Scaling types.

Type 1: Default property

The default behavior displays data in the respective chart format including the zero value. The graph is adjusted according to the data range available. Set the following property in the UI.Chart annotation:
<PropertyValue Property="AxisScaling" EnumMember="UI.ChartAxisScalingType/AdjustToDataIncluding0"/>

Type 2: Adjust scale property

The adjust scale property displays data in the respective chart format based on the data range avaiable. Set the following property in the UI.Chart annotation:
<PropertyValue Property="AxisScaling" EnumMember="UI.ChartAxisScalingType/AdjustToData"/>

Type 3: Min-Max property

The min-max property lets you define the minimum and maximum data range to display data in the respective chart format. Set the following property in the UI.Chart annotation: 

<PropertyValue Property="AxisScaling" EnumMember="UI.ChartAxisScalingType/MinMaxValues"/>
You must define the DataPoint annotation to set the minimum and maximum values.
<Annotation Term="UI.DataPoint" Qualifier="<DataPoint Qualifier>">

  <Record Type="UI.DataPointType">

    <PropertyValue Property="MinimumValue" Decimal="<Minimum Value>" />

    <PropertyValue Property="MaximumValue" Decimal="<Maximum Value>" />

    <PropertyValue Property="NumberFormat">

     <Record>

     .......

     </Record>

     </PropertyValue>

   </Record>

</Annotation>

Card Navigation

To configure navigation target on the KPI card, specify the navigation target (semanticObject, action and parameters) in the app-descriptor file (outbounds section of crossNavigation). To link this target, use the detailNavigation property within the KPI section in the app-descriptor.

While navigating from KPI card, the chart selection context is also passed in the navigation context.

The filter values in the chart selection context takes precedence over the filter values for the same dimension in the selection variant. Similarly, the parameter values specified in the selection variant takes precedence over the same parameter values specified in the app-descriptor (or in the crossNavigation parameters section).

Sample Code Cross Navigation
"crossNavigation": {
			"inbounds": {},
			"outbounds": {
				"ActualCostsKPIDetails": {
					"semanticObject": "ActualCosts",
					"action": "displayKPI",
					"parameters": {
						"companyName": "someName"
					}
				}
			}
		}
	},

Navigation to Local Object Page

Navigation from ALP to a local object page or sub-object page takes place when user appends the technical key of the object page entity set in the URL, or by configuring the SemanticKey annotation. Semantic keys take precedence over technical keys if both are present in the URL.

Local Object Page Navigation: If SalesOrder is the technical key for the entity set of the application, URL sample is as follows:
https://<server>:<port>/sap/bc/ui5_ui5/ui2/ushell/shells/abap/FioriLaunchpad.html#SalesOrder-displayFactSheet?SalesOrder=27

Sub-object Page Navigation: An object page may contain another object page. ALP allows direct navigation to the second level object page if deep linking is configured.

To configure deep linking set the allowDeepLinking to true in the descriptor file.
"pages": [
            {
                "entitySet": "ZCOSTCENTERCOSTSQUERY0020", // 1st level entity set (must be the same as the main entity set)
                "component": {
                        "name": "sap.suite.ui.generic.template.ObjectPage"
                 },
                "pages": [ //2nd level navigation
                            {
                                "navigationProperty": "to_CostElement",
                                "entitySet": "ZCOSTCENTERCOSTSQUERY0022", // 2nd level entity set
                                "component": {
                                    "name": "sap.suite.ui.generic.template.ObjectPage",
                                     "settings": {
                                           "allowDeepLinking": true
                                     }                                 
                                }
                            }
                        ]
                }
            ]
On configuring the descriptor file, users can navigate directly to the second level object page as shown in the following sample:
https://<server>:<port>/sap/bc/ui5_ui5/ui2/ushell/shells/abap/FioriLaunchpad.html#SalesOrder-displayFactSheet?SalesOrder=27&SupplierID=40

The SupplierID represents the semantic key of the second entity set.

SemanticKey based Navigation: Navigation based on SemanticKey annotation configuration at entity set level. The SemanticKey takes precedence over technical key if both are present in the URL.
<Annotation Term="com.sap.vocabularies.Common.v1.SemanticKey">
    <Collection>
        <PropertyPath>AccountingPrinciple</PropertyPath>
        <PropertyPath>CompanyCode</PropertyPath>
    </Collection>
</Annotation>

SelectionVariant Annotation

Annotation: com.sap.vocabularies.UI.SelectionVariant

You can also define a SelectionVariant annotation that contains filters and parameters to provide default values for the corresponding filter fields. This overrides the default values from the Common.FilterDefaultValue annotation. The filters are obtained from the SelectionVariant.SelectOptions and the parameters are obtained from the SelectionVariant.Parameters.

Properties:

  • SelectOptions

  • Parameters

<Annotation Term="UI.SelectionVariant" Qualifier="ActualCosts">
                    <Record>
                        <PropertyValue Property="SelectOptions">
                            <Collection>
                                <Record Type="UI.SelectOptionType">
                                    <PropertyValue Property="PropertyName">
                                        <PropertyPath>DisplayCurrency</PropertyPath>
                                    </PropertyValue>
                                    <PropertyValue Property="DisplayCurrency">
                                        <Collection>
                                            <Record Type="UI.SelectionRangeType">
                                                <PropertyValue EnumMember="UI.SelectionRangeSignType/I" Property="Sign"/>
                                                <PropertyValue EnumMember="UI.SelectionRangeOptionType/EQ" Property="Option"/>
                                                <PropertyValue Property="Low" String="EUR"/>
                                            </Record>
                                        </Collection>
                                    </PropertyValue>
                                </Record>                                
                            </Collection>
                        </PropertyValue>
                        <PropertyValue Property="Parameters">
                            <Collection>
                                <Record Type="UI.Parameter">
                                    <PropertyValue Property="PropertyName" PropertyPath="P_DisplayCurrency" />
                                    <PropertyValue Property="PropertyValue" String="EUR" />
                                </Record>
                            </Collection>
                        </PropertyValue>
                    </Record>
                </Annotation>
