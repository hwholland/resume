Creating Key Performance Indicator Tags

Key Performance Indicator (KPI) tag is an abbreviated and clickable title with a KPI value.

ALP supports a maximum of three KPIs. Each KPI can have its own OData source, entity set, and annotation file. The KPI value changes only if an action is executed on the transactional content. For example: Releasing sales orders affect a related KPI, or posting an accounting document affects certain financial KPI´s.

Hover over each action for a description. Click the action for more information.

Discriptor Settings: KPI Tag

Property: keyPerformanceIndicators

Description: Configuring this setting displays the KPI tag in the ALP title area. The parameters defined in the app-descriptor file points to a SelectionPresentationVariant qualifier associated with the KPI. ALP determines other related annotations from the SelectionPresentationVariant qualifier.

Configuration Sample:
"sap.ui.generic.app":{  
   "pages":[  
      {  
         "entitySet":"SEPMRA_C_ALP_SlsOrdItemCube",
         "component":{  
            "name":"sap.suite.ui.generic.template.AnalyticalListPage",
            "list":true,
            "settings":{  
               "keyPerformanceIndicators":{  
                  "KPIRevenue":{  
                     "model":"kpi",
                     "entitySet":"SEPMRA_C_ALP_TotalSalesKPI",
                     "qualifier":"KPIRevenue",
                     "detailNavigation":"OverviewPage"
                  }
            }
      }
},

Annotation: SelectionPresentationVariant with Qualifier="KPIRevenue"

Configuration Sample:

The SelectionPresentationVariant annotation qualifier relates to a SelectionVariant (optional) and PresentationVariant that provides a DataPoint annotation (for the KPI value). 

</Annotation>
  <Annotation Term="UI.SelectionPresentationVariant" Qualifier="KPIRevenue">
    <Record>
      <PropertyValue Property="Text" String="KPI: Net Sales by Calendar Quarter" />
      <PropertyValue Property="SelectionVariant" Path="@UI.SelectionVariant#KPIRevenue" />
      <PropertyValue Property="PresentationVariant" Path="@UI.PresentationVariant#KPIRevenue" />
    </Record>
</Annotation>
Sample SelectionVariant:
<Annotation Term="UI.SelectionVariant" Qualifier="KPIRevenue">
  <Record>
    <PropertyValue Property="Text" String="Default" />
  </Record>
</Annotation>

</Annotation>
  <Annotation Term="UI.PresentationVariant" Qualifier="KPIRevenue">

Annotation: DataPoint

Description: Configure this annotation to display KPI name, KPI value, scale factor, number of fractional digits, and criticality. 

<Annotation Term="UI.DataPoint" Qualifier="KPIValue">
  <Record>
    <PropertyValue Property="Value" Path="KPIValue" />
    <PropertyValue Property="Title" String="Sales Revenue" />
    <PropertyValue Property="Description" String="Sales revenue in the current quarter" />
    <PropertyValue Property="TargetValue" Path="KPITargetValue" />
    <PropertyValue Property="Visualization" EnumMember="UI.VisualizationType/Number" />
    <PropertyValue Property="ValueFormat">
      <Record>
        <PropertyValue Property="ScaleFactor" Decimal="1000000" />
        <PropertyValue Property="NumberOfFractionalDigits" Int="2" />
      </Record>
    </PropertyValue>
	<PropertyValue Property="Criticality" EnumMember="com.sap.vocabularies.UI.v1.CriticalityType/Positive"/>
        <PropertyValue Property="CriticalityCalculation">
          <Record>
            <PropertyValue Property="ImprovementDirection" EnumMember="UI.ImprovementDirectionType/Maximize" />
            <PropertyValue Property="DeviationRangeLowValue" Path="KPIDeviationRangeLowValue" />
            <PropertyValue Property="ToleranceRangeLowValue" Path="KPIToleranceRangeLowValue" />
          </Record>
        </PropertyValue>
        <PropertyValue Property="TrendCalculation">
          <Record>
            <PropertyValue Property="ReferenceValue" Path="KPITargetValue" />
            <PropertyValue Property="IsRelativeDifference" Bool="true" />
            <PropertyValue Property="UpDifference" Decimal="0.1" />
            <PropertyValue Property="StrongUpDifference" Decimal="1" />
            <PropertyValue Property="DownDifference" Decimal="0.1" />
            <PropertyValue Property="StrongDownDifference" Decimal="1" />
           </Record>
        </PropertyValue>
  </Record>
</Annotation>

KPI Value and Color

Use the SelectionPresentationVariant annotation to render KPI values and to determine the KPI's color. This annotation is associated with the SelectionVariant and PresentationVariant.

The first DataPoint visualization element in the PresentationVariant renders the KPI value and determines the KPI color. If a KPI is percent-based, then the “%” symbol appears along with the KPI value.
Note Click the KPI title or value to view more details on a KPI card.
The color of the KPI value depends on the threshold values. To define threshold values, use the following properties in the DataPoint annotation:
  • CriticalityCalculation: Allows you to hard code values or to include the value from a property path
  • Criticality: Allows complex back-end logic to specify the criticality values
Note If you use both properties in the annotation, then the criticality value overrides CriticalityCalculation value.

The coloring logic depends on the measure type. You can either choose a maximizing measure, minimizing measure, or range-based measure types based on the "ImprovementDirection" value of the measure.

  • Target Measure: Hover over each item to view the conditions.
  • Maximizing Measure: Hover over each item to view the conditions.
  • Minimizing Measure: Hover over each item to view the conditions.
If the threshold values are insufficient or incorrect, ALP chooses the closest color match for a KPI value. For example, if a target KPI has the DataPoint.Value aggregate < ThresholdValues.ToleranceRangeLowValue and does not have a value for ThresholdValues.DeviationRangeLowValue, then the KPI value has the color for critical.
Note The criticality indicator line in a KPI tag also takes the same color as the KPI value.

Scale and Decimal Precision

The SAPUI5 formatter returns the scale factor and decimal factor for a KPI value. You can specify the scale factor for KPI value by using the ScaleFactor property of the DataPoint annotation associated with the KPI. By default, ALP uses the scale factor returned by the SAPUI5 formatter when a KPI value is passed to it.
sap.ui.core.format.NumberFormat.getFloatInstance( {
style: "short", 
minFractionDigits: 0, 
maxFractionDigits: 1,
shortRefNumber: <scale>
} )
Percent-based KPI values are displayed with one decimal value, whereas a KPI value that is not percent-based measure is displayed with zero decimal points. However, in both cases the decimal values are rounded off respectively.
Note The sap:unit meta data annotation determines whether the KPI is percent-based.
You can override a unit of measure coming from backend. For example, the following annotation configuration overrides unit of measure coming from backend to percentage based.
<Annotations Target="CZ_PROJECTKPIS_CDS.CZ_PROJECTKPISType/ActualCosts">
	<Annotation Term="Org.OData.Measures.V1.Unit" String="%"/>
</Annotations>

SelectionVariant Annotation

Annotation: com.sap.vocabularies.UI.SelectionVariant

You can also define a SelectionVariant annotation that contains filters and parameters to provide default values for the corresponding filter fields. This overrides the default values from the Common.FilterDefaultValue annotation. The filters are obtained from the SelectionVariant.SelectOptions and the parameters are obtained from the SelectionVariant.Parameters.

Properties:

  • SelectOptions

  • Parameters

<Annotation Term="UI.SelectionVariant" Qualifier="ActualCosts">
                    <Record>
                        <PropertyValue Property="SelectOptions">
                            <Collection>
                                <Record Type="UI.SelectOptionType">
                                    <PropertyValue Property="PropertyName">
                                        <PropertyPath>DisplayCurrency</PropertyPath>
                                    </PropertyValue>
                                    <PropertyValue Property="DisplayCurrency">
                                        <Collection>
                                            <Record Type="UI.SelectionRangeType">
                                                <PropertyValue EnumMember="UI.SelectionRangeSignType/I" Property="Sign"/>
                                                <PropertyValue EnumMember="UI.SelectionRangeOptionType/EQ" Property="Option"/>
                                                <PropertyValue Property="Low" String="EUR"/>
                                            </Record>
                                        </Collection>
                                    </PropertyValue>
                                </Record>                                
                            </Collection>
                        </PropertyValue>
                        <PropertyValue Property="Parameters">
                            <Collection>
                                <Record Type="UI.Parameter">
                                    <PropertyValue Property="PropertyName" PropertyPath="P_DisplayCurrency" />
                                    <PropertyValue Property="PropertyValue" String="EUR" />
                                </Record>
                            </Collection>
                        </PropertyValue>
                    </Record>
                </Annotation>
