Design Time Aspects

You can upload and download files from and to the SAPUI5 ABAP repository with the following tools:

  • SAP Web IDE
  • SAPUI5 tools for Eclipse
  • SAPUI5 repository upload and download reports
Note

We recommend to use SAP Web IDE for developing complex apps with the latest innovations. SAPUI5 tools for Eclipse are only suitable for simple use cases.

When uploading files to the SAPUI5 ABAP repository, SAP Web IDE and SAPUI5 tools for Eclipse work differently:

SAP Web IDE

  • Uses Git for version control of the sources

  • Supports a client-side build that generates minified files and packages preload bundles to optimize start-up performance

  • Pushes the SAP Web IDE state to the ABAP system in a one-way, overwrite-all operation

SAPUI5 tools for Eclipse

  • Uses the team repository provider for version control of the sources

  • Sources are stored on the ABAP system and used simultaneously as deployment; build and packaging is not supported.

  • Uploading to the ABAP system is a two-way synchronization with conflict detection (based on touch timestamps).

Note

In Eclipse, you can also use Git for version control of the sources and the team repository provider for deployment. The ABAP system then only contains the deployment, but build and packaging is still not supported.

However, beware when using SAP Web IDE and Eclipse in parallel. In this case, you must always use Git for storing the sources. Also, we recommend that you deploy from SAP Web IDE only. Under no circumstances mix deployments from SAP Web IDE and Eclipse. Why? Each deployment from SAP Web IDE causes conflicts, as every file is touched (conflict detection uses touch timestamps and does not compare the actual content). Also, SAP Web IDE would deploy optimized content which the SAPUI5 tools for Eclipse would overwrite with non-optimized content.

As mentioned above, don't manually edit the BSP application representing a single SAPUI5 repository.

The SAPUI5 ABAP repository supports typical code pages for transferring of text files. However, there are some code pages that are not supported, for example code page CP932 containing Japanese characters. If in doubt, we recommend that you encode the files transferred to the SAPUI5 ABAP repository in UTF-8.
