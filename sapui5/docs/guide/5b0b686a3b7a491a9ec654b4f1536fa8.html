Action Control

You can use action control to display actions by adding the sap:action-for and sap:applicable-path term to your action or function import.

In the applicable path you specify a boolean property for the entity type which controls whether the function import can be invoked.

SAP Fiori Elements evaluates the applicable path and sets the visibility of the corresponding action based on the boolean property value.

If the condition as defined by the applicable-path variable is not fulfilled, the action is hidden or disabled as follows:

  • hidden if on page header level or for inline buttons on a list

  • disabled for header buttons of a list

If multi-select is available in a table, the action is allowed if at least one of the marked entries fulfils the condition.

In the following example, you can see the sap:action-for annotation which normally is generated implicitly by BOPF for any new action.

Sample Code Example from the Fiori Reference App Manage Products
<FunctionImport Name="SEPMRA_C_PD_ProductCopy" 
ReturnType="SEPMRA_PROD_MAN.SEPMRA_C_PD_ProductType" 
EntitySet="SEPMRA_C_PD_Product" m:HttpMethod="POST"
sap:action-for="SEPMRA_PROD_MAN.SEPMRA_C_PD_ProductType" sap:applicable-path="IsActiveEntity">
    <Parameter Name="ProductDraftUUID" Type="Edm.Guid" Mode="In"/>
    <Parameter Name="ActiveProduct" Type="Edm.String" Mode="In" MaxLength="10"/>
</FunctionImport>

If the action is defined then you must select an item or a line in a list. By default, these actions are disabled until the user selects at least one entity.
