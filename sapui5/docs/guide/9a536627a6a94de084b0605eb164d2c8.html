Configuration Options: Confirmation Popup and Save and Close Feature

In draft scenarios, you can enable UI elements to be displayed for specific situations.

Confirmation Popup

The system displays warning messages during the save process, for example, if fields have not been filled consistently or if entries are missing.

You can enable a confirmation popup that displays all warning messages, asking users if they still want to save the data.

To enable this popup, in the manifest.json under the object page settings, set the showConfirmationOnDraftActivate flag to true.

Save and Close Feature

When saving or activating the draft record, by default, the user stays on the object page. You can add a save and close logic to the Save button on the object page to enable users to navigate directly back to the list report. If the user clicks the button, the draft record is activated and the user navigates automatically to the list report page.

To enable this feature, in the manifest.json, set the navToListOnSave flag to true.

If the flag is set to true, the user automatically navigates to the list report page as soon as the draft record has been successfully activated.

If the flag is set to false or if the flag is not set at all, the user stays on object page.
