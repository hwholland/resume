Class sap.suite.ui.commons.MicroProcessFlowModule: sap/suite/ui/commons/MicroProcessFlow

extends prototypes
The MicroProcessFlow control can be used to track the progress of a process.
It is best used for linear processes, such as document, order, and approval flows, and can be embedded in tiles, tables, and other containers.

To display more complex processes and workflows, use sap.suite.ui.commons.ProcessFlow instead.

new sap.suite.ui.commons.MicroProcessFlow()

Constructor for a new MicroProcessFlow.

Accepts an object literal mSettings that defines initial property values, aggregated and associated objects as well as event handlers. See sap.ui.base.ManagedObject for a general description of the syntax of the settings object.

The supported settings are:

  • Properties

    • ariaLabel : string
    • renderType : sap.suite.ui.commons.MicroProcessFlowRenderType (default: Wrap)
    • width : sap.ui.core.CSSSize (default: )

  • Aggregations

    • content : sap.suite.ui.commons.MicroProcessFlowItem[]

In addition, all settings applicable to the base type sap.teaminstinct.prototypes can be used as well.

getRenderType(): sap.suite.ui.commons.MicroProcessFlowRenderType

Gets current value of property renderType.

Determines how the micro process flow should be rendered inside its parent container.
When the width of the parent container does not allow for all nodes in the micro process flow to be displayed, you can wrap it inside the parent container or add scrolling icons.

Default value is Wrap.
