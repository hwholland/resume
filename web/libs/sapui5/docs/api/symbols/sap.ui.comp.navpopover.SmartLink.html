Class sap.ui.comp.navpopover.SmartLinkModule: sap/ui/comp/navpopover/SmartLink

extends Link
The SmartLink control uses a semantic object to display NavigationPopover for further navigation steps.
Note: Navigation targets are determined using CrossApplicationNavigation of the unified shell service.

A detailed description of the OData annotations handled by this control is available here.
Annotation Limitations: Qualifier support for entity types is limited for this control.

new sap.ui.comp.navpopover.SmartLink(sId?, mSettings?)

Accepts an object literal mSettings that defines initial property values, aggregated and associated objects as well as event handlers. See sap.ui.base.ManagedObject for a general description of the syntax of the settings object.

The supported settings are:

  • Properties

    • semanticObject : string
    • additionalSemanticObjects : string[] (default: [])
    • semanticObjectController : any
    • fieldName : string
    • semanticObjectLabel : string
    • createControlCallback : object
    • mapFieldToSemanticObject : boolean (default: true)
    • contactAnnotationPath : string (default: undefined)
    • ignoreLinkRendering : boolean (default: false)
    • enableAvailableActionsPersonalization : boolean (default: true)
    • uom : string (default: undefined)

  • Aggregations

    • innerControl : sap.ui.core.Control

  • Events

    • beforePopoverOpens : fnListenerFunction or [fnListenerFunction, oListenerObject] or [oData, fnListenerFunction, oListenerObject]
    • navigationTargetsObtained : fnListenerFunction or [fnListenerFunction, oListenerObject] or [oData, fnListenerFunction, oListenerObject]
    • innerNavigate : fnListenerFunction or [fnListenerFunction, oListenerObject] or [oData, fnListenerFunction, oListenerObject]

In addition, all settings applicable to the base type sap.m.Link can be used as well.

Parameters:
stringsId?ID for the new control, generated automatically if no ID is given
objectmSettings?Initial settings for the new control

Contact(com.sap.vocabularies.Communication.v1)

Renders the contact information inside the NavigationPopover control. The Contact annotation contains various ContactType which can be used for providing additional information for distinguishing between the contact details. List of enumeration types supported is listed below:

  • com.sap.vocabularies.Communication.v1.PhoneType/fax: This enumeration type can be used for defining a fax number.
  • com.sap.vocabularies.Communication.v1.PhoneType/work: This enumeration type can be used for defining a work phone number.
  • com.sap.vocabularies.Communication.v1.PhoneType/cell: This enumeration type can be used for defining a mobile number.
  • com.sap.vocabularies.Communication.v1.ContactInformationType/work": This enumeration type can be used for defining a work email address.

XML Example of OData V4 with Contact Annotation

   <Annotations Target="EntityTypeName.Supplier"
     xmlns="http://docs.oasis-open.org/odata/ns/edm">
      <Annotation Term="com.sap.vocabularies.Communication.v1.Contact">
        <Record>
          <PropertyValue Property="fn" Path="FormattedName" />
          <PropertyValue Property="title" Path="Title" />
          <PropertyValue Property="org" Path="CompanyName" />
          <PropertyValue Property="role" Path="OrganizationRole" />
          <PropertyValue Property="photo" Path="Photo" />
          <PropertyValue Property="tel">
            <Collection>
              <Record>
                <PropertyValue Property="type"
                  EnumMember="com.sap.vocabularies.Communication.v1.PhoneType/fax" />
                <PropertyValue Property="uri" Path="FaxNumber" />
              </Record>
              <Record>
                <PropertyValue Property="type"
                  EnumMember="com.sap.vocabularies.Communication.v1.PhoneType/work />
                <PropertyValue Property="uri" Path="PhoneNumber" />
              </Record>
              <Record>
                <PropertyValue Property="type"
                  EnumMember="com.sap.vocabularies.Communication.v1.PhoneType/cell" />
                <PropertyValue Property="uri" Path="MobileNumber" />
              </Record>
            </Collection>
          </PropertyValue>
          <PropertyValue Property="email">
            <Collection>
              <Record>
                <PropertyValue Property="type"
                  EnumMember="com.sap.vocabularies.Communication.v1.ContactInformationType/work" />
                <PropertyValue Property="address" Path="EmailAddress" />
              </Record>
            </Collection>
          </PropertyValue>
        </Record>
     </Annotation>
   </Annotations>

The SmartLink control interprets the com.sap.vocabularies.UI.v1.Hidden annotation. Fields which are annotated as Bool: "true" are not shown in the contact information but the corresponding data is requested by the backend.

	<Annotations Target="EPM_DEVELOPER_SCENARIO_SRV.Supplier/Title" xmlns="http://docs.oasis-open.org/odata/ns/edm">
		<Annotation Term="com.sap.vocabularies.UI.v1.Hidden"/>
	</Annotations>
