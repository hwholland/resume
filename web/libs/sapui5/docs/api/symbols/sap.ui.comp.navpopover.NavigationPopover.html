Class sap.ui.comp.navpopover.NavigationPopoverModule: sap/ui/comp/navpopover/NavigationPopover

extends ResponsivePopover
The NavigationPopover control is used to present information in a specific format. Note: This control is used by the NavigationPopoverHandler and must not be used manually.

Constructor Detail
new sap.ui.comp.navpopover.NavigationPopover(sId?, mSettings?)

Constructor for a new navpopover/NavigationPopover.

Accepts an object literal mSettings that defines initial property values, aggregated and associated objects as well as event handlers. See sap.ui.base.ManagedObject for a general description of the syntax of the settings object.

The supported settings are:

  • Properties

    • semanticObjectName : string
    • semanticAttributes : object
    • appStateKey : string
    • mainNavigationId : string
    • availableActionsPersonalizationText : string (default: undefined)

  • Aggregations

    • availableActions : sap.ui.comp.navpopover.LinkData[]
    • mainNavigation : sap.ui.comp.navpopover.LinkData
    • ownNavigation : sap.ui.comp.navpopover.LinkData

  • Associations

    • source : (sap.ui.core.ID | sap.ui.core.Control)
    • extraContent : (sap.ui.core.ID | sap.ui.core.Control)
    • component : (sap.ui.core.ID | sap.ui.core.Element)

  • Events

    • targetsObtained : fnListenerFunction or [fnListenerFunction, oListenerObject] or [oData, fnListenerFunction, oListenerObject]
    • navigate : fnListenerFunction or [fnListenerFunction, oListenerObject] or [oData, fnListenerFunction, oListenerObject]
    • availableActionsPersonalizationPress : fnListenerFunction or [fnListenerFunction, oListenerObject] or [oData, fnListenerFunction, oListenerObject]

In addition, all settings applicable to the base type sap.m.ResponsivePopover can be used as well.

Parameters:
stringsId?ID for the new control, generated automatically if no ID is given
objectmSettings?Initial settings for the new control
