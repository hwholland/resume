Class sap.m.ResponsivePopoverModule: sap/m/ResponsivePopover

extends Control
known direct subclasses: NavigationPopover

Overview

The responsive popover acts as a popover on desktop and tablet, while on phone it acts as a dialog with stretch set to true.

Note: It is recommended that ResponsivePopover is used in fragments otherwise there might be some implications on the user experience. For example, on desktop, open or close functions of the Popover might not be called.

Usage

When you want to make sure that all content is visible on any device.


Since: 1.15.1.

Constructor Detail
new sap.m.ResponsivePopover(sId?, mSettings?)

Constructor for a new ResponsivePopover.

Accepts an object literal mSettings that defines initial property values, aggregated and associated objects as well as event handlers. See sap.ui.base.ManagedObject for a general description of the syntax of the settings object.

The supported settings are:

  • Properties

    • placement : sap.m.PlacementType (default: Right)
    • showHeader : boolean (default: true)
    • title : string
    • icon : sap.ui.core.URI
    • modal : boolean
    • offsetX : int
    • offsetY : int
    • showArrow : boolean (default: true)
    • contentWidth : sap.ui.core.CSSSize
    • contentHeight : sap.ui.core.CSSSize
    • horizontalScrolling : boolean (default: true)
    • verticalScrolling : boolean (default: true)
    • showCloseButton : boolean (default: true)
    • resizable : boolean (default: false)

  • Aggregations

    • content : sap.ui.core.Control[]
    • customHeader : sap.m.IBar
    • subHeader : sap.m.IBar
    • beginButton : sap.m.Button
    • endButton : sap.m.Button

  • Associations

    • initialFocus : (sap.ui.core.ID | sap.ui.core.Control)
    • ariaLabelledBy : (sap.ui.core.ID | sap.ui.core.Control)[]
    • ariaDescribedBy : (sap.ui.core.ID | sap.ui.core.Control)[]

  • Events

    • beforeOpen : fnListenerFunction or [fnListenerFunction, oListenerObject] or [oData, fnListenerFunction, oListenerObject]
    • afterOpen : fnListenerFunction or [fnListenerFunction, oListenerObject] or [oData, fnListenerFunction, oListenerObject]
    • beforeClose : fnListenerFunction or [fnListenerFunction, oListenerObject] or [oData, fnListenerFunction, oListenerObject]
    • afterClose : fnListenerFunction or [fnListenerFunction, oListenerObject] or [oData, fnListenerFunction, oListenerObject]

In addition, all settings applicable to the base type sap.ui.core.Control can be used as well.

Parameters:
stringsId?ID for the new control, generated automatically if no ID is given
objectmSettings?Initial settings for the new control A popover-based control that behaves differently according to the device it is on.

Opens the ResponsivePopover. The ResponsivePopover is positioned relatively to the control parameter on tablet or desktop and is full screen on phone. Therefore the control parameter is only used on tablet or desktop and is ignored on phone.
Parameters:
objectoParentWhen this control is displayed on tablet or desktop, the ResponsivePopover is positioned relative to this control.
Returns:
objectReference to the opening control
