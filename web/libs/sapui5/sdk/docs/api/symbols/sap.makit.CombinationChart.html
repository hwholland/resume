Constructor Detail
new sap.makit.CombinationChart(sId?, mSettings?)
Constructor for a new CombinationChart.

Accepts an object literal mSettings that defines initial property values, aggregated and associated objects as well as event handlers.

If the name of a setting is ambiguous (e.g. a property has the same name as an event), then the framework assumes property, aggregation, association, event in that order. To override this automatic resolution, one of the prefixes "aggregation:", "association:" or "event:" can be added to the name of the setting (such a prefixed name must be enclosed in single or double quotes).

The supported settings are:

  • Properties
    • width : sap.ui.core.CSSSize (default: '100%')
    • height : sap.ui.core.CSSSize (default: '100%')
    • showRangeSelector : boolean (default: true)
    • legendPosition : sap.makit.LegendPosition (default: sap.makit.LegendPosition.Left)
    • primaryColorPalette : any
    • secondaryColorPalette : any
    • showTableValue : boolean (default: true)
    • numberOfVisibleCategories : int
    • rangeSelectorStartPosition : int (default: 0)
  • Aggregations
    • categoryRegions : sap.makit.Category[]
    • layers : sap.makit.Layer[]
    • categoryAxis : sap.makit.CategoryAxis
    • primaryValueAxis : sap.makit.ValueAxis
    • secondaryValueAxis : sap.makit.ValueAxis
    • valueBubble : sap.makit.ValueBubble
  • Associations
    • Events
      • doubletap : fnListenerFunction or [fnListenerFunction, oListenerObject] or [oData, fnListenerFunction, oListenerObject]
      • tap : fnListenerFunction or [fnListenerFunction, oListenerObject] or [oData, fnListenerFunction, oListenerObject]
      • longpress : fnListenerFunction or [fnListenerFunction, oListenerObject] or [oData, fnListenerFunction, oListenerObject]
    Parameters:
    {string} sId? id for the new control, generated automatically if no id is given
    {object} mSettings? initial settings for the new control
