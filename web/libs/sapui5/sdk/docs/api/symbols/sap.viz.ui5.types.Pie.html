new sap.viz.ui5.types.Pie(sId?, mSettings?)
Constructor for a new ui5/types/Pie.

Accepts an object literal mSettings that defines initial property values, aggregated and associated objects as well as event handlers.

If the name of a setting is ambiguous (e.g. a property has the same name as an event), then the framework assumes property, aggregation, association, event in that order. To override this automatic resolution, one of the prefixes "aggregation:", "association:" or "event:" can be added to the name of the setting (such a prefixed name must be enclosed in single or double quotes).

    • isGeoPie : boolean (default: false)
    • valign : sap.viz.ui5.types.Pie_valign (default: sap.viz.ui5.types.Pie_valign.top)
    • drawingEffect : sap.viz.ui5.types.Pie_drawingEffect (default: sap.viz.ui5.types.Pie_drawingEffect.normal)
    • formatRules : object[]
    • plotScale : float (default: 1)

      In addition, all settings applicable to the base type sap.viz.ui5.core.BaseStructuredType can be used as well.

      getFormatRules() : object[]
      Getter for property formatRules. Rules to format data points, sample: [{condition: [{Key1:"Value1", Key2:"Value2"}], color:"#00ff00"}, {condition: [{Key3:"Value3"}], color:"#00ffff"}]. Each rule has two properties: "condition" and "color". The relation among the condition object in "condition" array is "OR", which means the data point that met any condition in the array will apply the "color". If multiple rules could apply on the same data point, the last rule would take effect.

      Default value is empty/undefined

      Returns:
      {object[]} the value of property formatRules
