JavaScript Namespaces

To avoid conflicts with other frameworks or development, the JavaScript development follows the OpenAJAX concept for namespaces. For this, the sap namespace has been officially reserved for SAP.

All objects, functions, classes, and so on, that have been developed by SAP and are visible globally must either be defined as direct members of that namespace, or as members of one of its subnamespaces. Non-OpenUI5 content, such as application code or controls developed by customers and partners, must not use this sap namespace prefix.

For OpenUI5-related development, the namespace is further restricted to the sap.ui child namespace with an exception for the sap.m entities.

Note

SAP's JavaScript objects with an accessible name that are not part of a closure must have a name starting with the reserved namespace sap. The namespace for OpenUI5 objects must start with sap.ui or sap.m.

Global variables, for example window.xyz, are forbidden. They may lead to conflicts with other frameworks, applications, and so on.

Restriction
If you develop own artifacts that are not owned by SAP, you must not use the following namespace prefixes:

  • sap* as it is reserved for SAP

  • src*for technical reasons

To ease the handling of namespaces, OpenUI5 the helper function jQuery.sap.getObject. For more information, see API Reference: jQuery.sap.getObject.

Example
// A declaration, ensures that sap.ui.sample exists
jQuery.sap.getObject("sap.ui.sample.myNamespace", 0);
 
// now it is safe to use that namespace object and assign a new member 'myNamespace'
// to it
// So the developer can decide whether it should be a function, an object,
// or an instance of a specific type
sap.ui.sample.myNamespace= { key1 : 'value1' };
 
// the following line guarantees that <code>sap.ui.sample.mySubspace</code>
// is a valid object
jQuery.sap.getObject("sap.ui.sample.mySubspace", 0);
 
// now one can use this namespace as well
sap.ui.sample.mySubspace.member1 = 42;
sap.ui.sample.mySubspace.member2 = 3.141;

Naming of Subnamespaces (Packages)

Subnamespaces of sap.ui that do not represent classes should have a lowercase name like the Java packages whereas CamelCase should be avoided. 

  sap.ui.base
  sap.ui.core
  sap.ui.util
